# 🍜 Order By QR

This folder contains instructions and scripts necessary to train and integrate the AI ​​model for classifying food images (YOLOv8) into the Order By QR system. This AI model allows the Admin page to automatically recognize images and intelligently fill in information such as Dish Name, Price, and Category.

## 🚀 Training & Integration Guide

To save personal computer resources and leverage GPU capabilities, the training process will be fully automated on **Google Colab**, then the model will be exported as **TensorFlow.js** for offline operation directly in the browser (frontend).

### Step 1: Data Management (Roboflow)
1. Access [Roboflow](https://roboflow.com/) to upload, label, and enhance your food image data.

2. Create a Dataset Version. Note: Ensure a sufficient variety of images for each item to increase accuracy.

3. Update the `API_KEY` code and Workspace/Project information in the script (when using a different project).

### Step 2: Training (Google Colab)
1. Open [Google Colab](https://colab.research.google.com/) and upload the script file [`train_classifier.py`](./scripts/train_classifier.py).

2. Change the Runtime to **GPU (T4)** to increase speed.

3. Run the entire script. The system will automatically:

- Fetch the dataset directly from Roboflow.

- Restructure the data in preparation for YOLOv8 classification.

- Conduct training on the model (`yolov8n-cls.pt`).

- Directly export the standard weights to TensorFlow.js format (the lightest compressed format for web use).

- Package (zip) the results into a file named `model_for_frontend.zip` in the Files column on the left for you to download.

### Step 3: Integrate into Frontend
1. Download the compressed file `model_for_frontend.zip` from Colab to your computer.

2. Extract all the files inside and copy them to the frontend at the following path:

`frontend/public/models/dish-classifier/`
3. The files you receive will include the AI ​​network (e.g., `model.json` and the `group*-shard*of*.bin` files).

4. **Extremely important:** The raw model does not include prices and item names. You must create and configure the `labels.json` file in the same directory to provide metadata for the autofill function.

**Standard format of `labels.json`:**

```json
{

"0": { "name": "7 Up", "categoryid": 6, "price": 12000 },

"1": { "name": "Stir-fried Corn", "categoryid": 3, "price": 20000 }
}
```
*Note: The ID keys ("0"`, "1"`) generated by YOLO depend on the class order during training. Make sure you match the key exactly with the corresponding dish.*
